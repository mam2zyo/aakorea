# A.A. KOREA 웹앱 도메인 명세서: Member

공통 아키텍처 원칙과 모듈 간 공통 규칙은 [DOMAIN_SPEC.md](./DOMAIN_SPEC.md)를 따른다.
이 문서는 `member` 도메인만의 책임 범위와 외부 도메인 계약만 정의한다.

## 1. 책임 범위

`member`는 시스템 전반의 사용자 식별과 공통 계정 마스터를 담당한다.
이 도메인이 직접 소유하는 핵심 엔티티는 다음 두 가지다.

- `Member`
- `DeliveryAddress`

다음 책임은 `member`에 포함하지 않는다.

- 주문, 주문 이력, 결제 스냅샷
- 구독, 발송 이력, 수령인 스냅샷
- 그룹 운영 권한의 최종 판정
- 지역연합 공개 디렉터리, 사이트 모음, 역사/연혁, 공개 자료처럼 로그인 없이 읽는 정보 허브

---

## 2. 핵심 기능

### 2.1. 인증과 계정 상태

- 공통 로그인 처리 (`OIDC`, `OAuth2` 연계 가능)
- 내부 세션 또는 접근 토큰 발급
- 계정 상태 조회

### 2.2. 최소 프로필 관리

- 기본 프로필 정보 관리
- 연락처 관리
- 공통 역할, 상태 조회

### 2.3. 주소록 마스터 관리

- 기본 배송지 설정
- 다중 배송지 저장
- 주소 수정, 삭제

주소록은 `store`, `heart`가 스냅샷을 만들기 전에 참조하는 원본 데이터다.

---

## 3. 도메인 규칙

- `member`는 공통 로그인과 최소 계정 마스터만 담당한다.
- 주문, 구독, 발송처럼 거래 시점의 기록은 각 거래 도메인이 자체 스냅샷으로 보존한다.
- 주소록이 바뀌더라도 과거 주문, 구독 기록은 변경되지 않는다.
- 로그인 이후 실제 권한은 `memberId`를 기준으로 각 도메인 유스케이스에서 다시 판정한다.
- 도메인별 운영 자격은 `member` 공통 역할만으로 모두 표현하지 않고, 각 도메인 정책에서 관리한다.
- 레거시 메뉴명이나 페이지명에 `members`가 포함되더라도, 공개 정보 허브 성격의 페이지는 `member` 도메인으로 보지 않는다.

---

## 4. 외부 도메인 계약

### 4.1. `service`

- 운영 계정과 `memberId`를 연결해 GSR 배정, 그룹 변경 권한, 공개 콘텐츠 운영 자격을 판정한다.
- `member`는 계정 식별과 상태를 제공하고, 운영 권한 자체는 소유하지 않는다.

### 4.2. `store`

- 주문 생성 직전에 `memberId` 기준으로 배송지 원본을 조회한다.
- 결제 이후 배송지 정보는 `store`가 자체 스냅샷으로 보존한다.

### 4.3. `heart`

- 구독 생성 시 `buyerMemberId`로 결제자를 식별한다.
- 실제 수령인 정보는 `heart`가 자체 스냅샷으로 저장한다.

### 4.4. 통합 조회

마이페이지나 관리자 포털은 여러 도메인 데이터를 집계해 보여줄 수 있지만, 조회 통합이 곧 쓰기 책임 통합을 의미하지는 않는다.

---

## 5. 구현 초점

- 공통 로그인 연계와 계정 상태 판정 안정성
- `memberId` 기반 권한 연결
- 기본 배송지 지정과 다중 배송지 일관성
- 다른 도메인의 식별자 기반 조회 계약 유지

---

## 6. 후속 상세화 항목

- 회원가입 정책과 가입 절차
- 외부 IdP 연동 상세 프로토콜
- 도메인별 운영 권한 할당 세부 정책
- 프로필 항목 확장 범위와 개인정보 처리 정책
