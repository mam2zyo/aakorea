# aa.org 디자인 분석 및 AAKorea 적용 가이드

작성일: 2026-03-26

## 1. 목적

이 문서는 `aa.org`의 홈 화면 디자인 요소를 분석하고, `AAKorea` 프론트엔드에 적용 가능한 형태로 재구성하기 위해 작성했다.

- 대상 범위: 데스크톱/모바일 홈 화면, 헤더, 히어로, CTA, 리소스 링크 리스트, 푸터
- 적용 대상: `frontend/aakorea/src`의 퍼블릭 영역
- 목표: "그대로 복제"가 아니라, 신뢰감과 회복 중심의 분위기를 우리 앱 맥락에 맞게 번역

## 2. 분석 기준

분석은 아래 자료를 기준으로 진행했다.

- 공식 사이트: https://www.aa.org/
- 2026-03-26 기준 홈 화면 HTML/CSS 확인
- 사용자가 제공한 데스크톱/모바일 스크린샷

참고:

- 실제 HTML에서 홈 화면의 주요 섹션 구조와 CTA 라벨을 확인했다.
- 실제 CSS에서 브랜드 색상, 기본 폰트 스택, 반응형 그리드 관련 값을 확인했다.
- 헤드라인용 serif 폰트는 CSS상 `crete` 계열로 보인다. 이는 실제 렌더링 기준 추정이 아니라 CSS 선언 기준이다.

## 3. 한 줄 요약

`aa.org`의 디자인은 "공공기관 수준의 신뢰감 + 공동체의 따뜻함 + 누구나 바로 행동할 수 있는 단순한 안내"에 집중되어 있다.

핵심 성격은 다음과 같다.

- 시각적으로 안정적이다.
- 정보 구조가 명확하다.
- 강조색은 적지만 분명하다.
- 감성보다 안내와 안심을 우선한다.
- 모바일에서도 메시지와 CTA가 흐트러지지 않는다.

## 4. 시각 시스템 핵심

### 4.1 색상

실제 CSS에서 확인된 핵심 팔레트는 아래와 같다.

| 역할 | 값 | 설명 | AAKorea 적용 제안 |
| --- | --- | --- | --- |
| Brand Blue | `#3977C3` | 메인 브랜드 컬러, 헤더/푸터/강조 블록 | 퍼블릭 헤더, 메인 CTA 배경, 푸터 메인 배경 |
| Deep Blue | `#164580` | 더 강한 액션/명도 대비 | 활성 버튼, hover, 중요한 링크 강조 |
| Accent Orange | `#E9A151` | 주목용 강조색 | 공지 배너, 보조 CTA, 하이라이트 |
| Sky Accent | `#26A3DD` | hover/interactive accent | 링크 hover, 아이콘 active 상태 |
| Light Gray | `#E9E9E9` | 배경 구분/중립 면 | 탭, 분할 배경, 입력 필드 배경 |
| Border Gray | `#DBDBDC` | 경계선 | 리스트 divider, 카드 border |
| Text Dark | `#222222` | 기본 본문색 | 기본 텍스트 |
| Text Muted | `#646464` | 보조 본문색 | 캡션, 보조 설명 |
| Focus | `#FFCC4E` | 포커스 링 | 접근성 focus outline |

색상 운용 방식의 특징:

- 파란색은 "브랜드"이자 "신뢰"의 색으로 넓은 면적에 과감하게 사용한다.
- 오렌지는 넓게 쓰지 않고, 반드시 행동을 유도하거나 주의를 주는 지점에만 쓴다.
- 전체 배경은 대부분 흰색 또는 밝은 회색으로 유지해 콘텐츠를 우선시한다.

### 4.2 타이포그래피

실제 CSS 기준:

- 본문/네비게이션: `Roboto`, sans-serif
- 헤드라인 계열: `crete` 계열 serif

시각적 역할 분리:

- 큰 제목은 serif로 신뢰감과 상징성을 준다.
- 본문과 기능 UI는 sans-serif로 읽기 편하게 유지한다.
- 제목은 크고 무겁지만, 문장 수는 짧다.
- 본문은 설명적이지만 문단 길이를 과하게 늘리지 않는다.

AAKorea 적용 제안:

- 헤드라인: `Noto Serif KR`, `Nanum Myeongjo`, serif
- 본문/UI: `Pretendard`, `Noto Sans KR`, sans-serif

이유:

- 한글 환경에서 `Roboto + Crete`를 그대로 쓰기보다, 현재 코드베이스와 가독성에 맞는 한글 대체가 안정적이다.
- 현재 `src/index.css`가 이미 `Noto Sans KR`, `Pretendard`, `Noto Serif KR` 계열을 사용 중이라 충돌이 적다.

### 4.3 그리드와 여백

HTML/CSS 기준으로 확인되는 구조적 특징:

- 퍼블릭 컨테이너 폭은 약 `1180px` 수준
- 데스크톱 중심 12-column 리듬
- 태블릿/모바일에서는 8-column 또는 4-column 감각으로 축소
- 섹션 사이 간격이 넉넉하고, 각 블록의 역할이 시각적으로 명확하다

적용 원칙:

- 페이지 최대 너비는 현재 `PublicShell`의 `max-w-[1180px]`를 유지해도 좋다.
- 대신 내부 카드 간격과 섹션 구분을 더 평면적이고 구조적으로 정리하는 편이 `aa.org` 감성에 가깝다.

## 5. 컴포넌트 패턴 추출

### 5.1 헤더

패턴:

- 파란 배경의 고정감 있는 브랜드 헤더
- 좌측 로고, 우측 유틸리티 링크
- 그 아래 또는 같은 줄에 메인 네비게이션
- 검색, 언어 전환, 지역 찾기 같은 기능성 요소가 잘 보인다

느낌:

- 마케팅 사이트보다 "공식 안내 포털"에 가깝다

AAKorea 적용 포인트:

- 현재 `PublicShell.jsx`의 반투명 유리 헤더를, 보다 명확한 솔리드 블루 헤더로 전환
- 상단 유틸리티 영역과 메인 네비게이션 영역을 분리
- `내 주변 모임 찾기`, `처음 오셨나요`, `GSO 안내` 같은 빠른 액션을 상단에 노출
- 메인 헤더에는 내부 서비스 메뉴만 남기고, 네이버 블로그/유튜브/카카오톡 같은 외부 채널은 `공식 채널` 묶음 또는 유틸리티 메뉴로 분리
- `English` 같은 언어 전환은 메인 네비게이션이 아니라 유틸리티 링크로 배치
- 실제 운영하지 않는 채널은 메인 헤더에서 제거하거나 우선순위를 낮춘다
- 카카오톡 플러스친구처럼 운영 정책이 불명확한 채널은 리뉴얼 시점에는 전면 CTA로 두지 않고, 운영 책임과 활용 목적이 정해진 뒤 재도입한다

### 5.2 히어로

패턴:

- 넓은 이미지 배경
- 어두운 오버레이 위에 짧고 강한 메시지
- 왼쪽에는 공감형 카피와 CTA
- 오른쪽에는 기능 카드 또는 탐색 카드

핵심:

- 히어로가 단순 장식이 아니라 "첫 행동"을 유도한다

AAKorea 적용 포인트:

- 메인 홈의 첫 화면에 `모임 찾기`, `처음 방문 가이드`, `A.A. 소개` 중 하나를 즉시 선택할 수 있게 배치
- 데스크톱에서는 오른쪽 보조 카드 유지
- 모바일에서는 보조 카드를 접고 주 CTA만 남김

### 5.3 공지/배너

패턴:

- 히어로 아래에 전폭(full-width) 안내 배너
- 오렌지 배경과 진한 텍스트로 즉시 주목

장점:

- 긴급 안내, 운영 변경, 주요 일정 공지에 적합

AAKorea 적용 포인트:

- `GSO 운영 공지`, `사무실 방문 안내`, `대회/컨퍼런스 공지`, `문헌 배송 안내` 등에 사용
- 홈과 GSO 페이지에서 재사용 가능한 `NoticeBanner` 컴포넌트로 분리

### 5.4 페르소나 전환 바

패턴:

- `I Am...`처럼 사용자 상황을 빠르게 선택하게 하는 세그먼트/셀렉터
- 선택에 따라 바로 아래 콘텐츠 영역이 달라진다

장점:

- 초방문자, 가족, 전문가, 기존 멤버처럼 목적이 다른 사용자를 빠르게 분기할 수 있다

AAKorea 적용 포인트:

- 홈 화면에 `처음 오신 분`, `가족/지인`, `전문가`, `기존 멤버` 전환 바 적용
- 라우팅 분기보다 홈에서 정보 구조를 먼저 정리하는 수단으로 쓰는 것이 좋다

### 5.5 분할형 콘텐츠 블록

패턴:

- 좌우 2단 블록
- 한쪽은 비디오/이미지, 다른 쪽은 파란 배경 설명 패널
- CTA는 라운드 pill 버튼

장점:

- 감정적 공감과 정보 전달을 동시에 처리

AAKorea 적용 포인트:

- `A.A.는 어떤 모임인가`, `첫 모임은 어떻게 진행되나`, `GSO는 어떤 일을 하나`에 적합
- `HomePage.jsx`, `GsoPage.jsx`에서 우선 적용 가능

### 5.6 리소스 링크 리스트

패턴:

- 제목 + 세로형 링크 목록
- 항목 사이 구분선
- 오른쪽에 chevron 또는 화살표

장점:

- 문서/가이드/FAQ 탐색이 쉬워진다

AAKorea 적용 포인트:

- `처음 방문 가이드`, `자주 묻는 질문`, `12단계/12전통`, `모임 형태 설명`, `GSO 자료`를 묶어서 제공
- 카드보다 리스트가 더 어울리는 정보형 링크는 이 패턴으로 통일

### 5.7 Daily Reflections 타입 블록

패턴:

- 좌측 타이틀 영역
- 우측 캘린더 또는 날짜 선택 인터랙션
- 선택 날짜에 따라 본문이 바뀌는 구조

AAKorea 적용 포인트:

- 현재 바로 필요하지 않더라도 `오늘의 묵상`, `오늘의 공지`, `일정 캘린더` 컴포넌트로 확장 가능
- 구조적으로는 `SectionHeader + SideControl + ContentPanel` 조합으로 일반화 가능

### 5.8 CTA 밴드

패턴:

- 넓은 파란색 밴드
- 큰 serif 제목
- 오른쪽 또는 하단에 2개 CTA 버튼

효과:

- 페이지 하단에서 다시 한 번 행동을 유도한다

AAKorea 적용 포인트:

- `가까운 모임 찾기`, `처음 방문 가이드 보기`
- `GSO 문의하기`, `모임 등록 절차 보기`

### 5.9 푸터

패턴:

- 상단: 뉴스레터/보조 안내
- 중단: 큰 브랜드명 + pill 링크들
- 하단: 텍스트 링크와 법적 문구
- `Back to Top` 원형 버튼

AAKorea 적용 포인트:

- 현재 `SiteFooter.jsx`를 더 구조화해서 3단 푸터로 개편
- 주요 문헌/공지/앱/연락처를 pill 링크로 묶기
- 긴 법적 문구나 운영 원칙은 하단 작은 텍스트 영역으로 분리

## 6. 모바일 패턴

모바일 스크린샷 기준으로 보이는 중요한 특징:

- 상단 네비게이션은 아이콘 중심으로 축소
- 히어로의 오른쪽 기능 카드는 사라지거나 뒤로 밀린다
- CTA 버튼은 넓고 명확하게 유지된다
- 2단 분할 블록은 단일 컬럼으로 순차 정렬된다
- 링크 리스트는 카드보다 리스트형 구조가 더 강하게 드러난다
- 푸터의 pill 링크들은 여러 줄로 잘 감긴다

AAKorea 적용 원칙:

- 모바일에서는 "탐색보다 행동"을 우선
- 보조 정보는 아래로 보내고, 첫 버튼은 항상 화면 초반에 노출
- 가로 배치보다 세로 배치 안정성을 우선

## 7. 현재 AAKorea와의 차이

현재 퍼블릭 테마(`frontend/aakorea/src/index.css`)는 아래 성향이 강하다.

- 베이지 기반의 따뜻한 배경
- glassmorphism 성격
- 카드 위주의 분절된 UI
- 곡선형 카드와 부드러운 그림자

반면 `aa.org`는 아래 성향이 강하다.

- 파란색과 흰색 중심의 공공기관형 브랜드 톤
- blur보다 flat block 중심
- 구조적 구획이 명확한 안내형 레이아웃
- 카드보다 밴드, 패널, 리스트, 블록을 더 적극적으로 사용

정리하면:

- 현재 테마는 "따뜻하고 부드러운 안내"
- `aa.org`는 "공식적이고 신뢰감 있는 안내"

따라서 전체를 한 번에 바꾸기보다, 퍼블릭 정보 영역에만 `aa.org` 패턴을 우선 도입하는 편이 안전하다.

## 8. 적용 전략

### 8.1 그대로 가져와도 좋은 것

- 파란색 기반의 브랜드 구조
- serif headline + sans body 조합
- pill CTA 버튼
- 전폭 공지 배너
- 사용자 상황별 분기 바
- 링크 리스트형 정보 구조
- 강한 푸터 구조

### 8.2 우리 식으로 번역해야 하는 것

- 폰트는 한글 가독성 기준으로 대체
- 이미지와 일러스트는 국내 맥락에 맞게 재제작
- 미국 사이트의 직접 문구는 한국형 정보 구조로 재서술
- 지역 찾기/모임 찾기 UI는 우리 데이터 구조에 맞게 설계
- 외부 채널은 브랜딩 요소로 흩뿌리기보다, 운영 중인 채널만 선별해 묶음으로 제공

### 8.3 가져오면 안 되는 것

- `Alcoholics Anonymous` 로고 자산 자체
- `Blue People` 그래픽 같은 등록 상표 비주얼
- 사이트 카피의 직접 복제
- 구조는 유사하지만 정보 정확성이 다른 미국 기준 콘텐츠
- 관리되지 않는 외부 채널을 "주요 행동"처럼 전면 노출하는 정보 구조

## 11. 구현 우선순위

### 1단계

## 12. 권장 구현 방향

가장 좋은 방향은 "현재의 따뜻한 AAKorea 분위기"를 완전히 버리지 않고, `aa.org`의 구조적 장점을 입히는 것이다.

추천 방향:

- 퍼블릭 shell은 `aa.org`처럼 더 공식적이고 명료하게
- 카드 내부 온도감은 현재 AAKorea의 부드러움을 일부 유지
- 블루는 구조용, 오렌지는 행동용으로 역할을 분명히 분리
- 관리자 영역에는 이 스타일을 바로 확장하지 말고, 퍼블릭 영역에서 먼저 검증

## 13. 결론

`aa.org`에서 배워야 할 가장 중요한 포인트는 "브랜드 색"이 아니라 "정보를 안심시키는 방식"이다.

우리 앱에 적용할 때 핵심은 아래 세 가지다.

- 첫 화면에서 바로 행동할 수 있어야 한다.
- 누구를 위한 정보인지 빠르게 구분되어야 한다.
- 페이지가 많아져도 헤더, CTA, 링크 리스트, 푸터 패턴이 일관되어야 한다.

즉, `aa.org` 스타일 적용은 단순한 리디자인이 아니라 퍼블릭 정보 구조의 재정렬로 이해하는 것이 맞다.
