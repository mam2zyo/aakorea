# A.A. KOREA 웹앱 도메인 명세서: Store

공통 아키텍처 원칙과 모듈 간 공통 규칙은 [DOMAIN_SPEC.md](./DOMAIN_SPEC.md)를 따른다.
이 문서는 `store` 도메인만의 거래 모델과 운영 규칙만 정리한다.

## 1. 책임 범위

`store`는 A.A. 공식 교본, 팸플릿 등 실물 문헌을 판매하는 단발성 이커머스 도메인이다.
향후 `store.aakorea.org`로 분리 가능한 구조를 목표로 한다.

이 도메인이 직접 소유하는 핵심 엔티티는 다음과 같다.

- `Item`
- `Cart`
- `CartItem`
- `Order`
- `OrderItem`

---

## 2. 핵심 기능

### 2.1. 상품 및 재고 관리

- 상품 코드 관리 예: `B-1`
- 카테고리 관리
- 가격 관리
- 재고 수량(`stockQuantity`) 관리
- 판매 가능 여부 관리

재고는 실물 기준이며, 디지털 상품은 기본 범위에 포함하지 않는다.

### 2.2. 장바구니 및 온라인 주문

- 사용자는 장바구니에 상품을 담는다.
- 주문 생성 시 장바구니 항목을 주문 항목으로 확정한다.
- PG 결제를 통해 `ONLINE` 주문을 생성한다.

### 2.3. 오프라인 주문 등록

관리자는 행사, 현장 판매 등 오프라인 판매를 `OFFLINE` 채널로 직접 등록할 수 있다.

- 즉시 재고 차감
- 결제 완료 상태로 시작 가능
- 별도 배송이 없는 경우 수령 완료 처리 가능

### 2.4. 주문 상태와 배송 처리

대표 상태 예시는 다음과 같다.

- `PENDING_PAYMENT`
- `PAID`
- `SHIPPED`
- `CANCELLED`

운영자는 `PAID` 주문을 기준으로 배송지 정보를 엑셀로 추출하고, 발송 후 `SHIPPED`로 변경한다.

---

## 3. 도메인 규칙

- 주문 생성 시 배송지와 가격은 스냅샷으로 고정한다.
- `Order`는 결제 시점의 배송지 정보를 보존한다.
- `OrderItem`은 결제 시점의 상품명, 상품 코드, 가격을 보존한다.
- 결제 완료 이후 주소록 마스터 변경이 기존 주문을 바꾸지 않는다.
- 결제 완료 이후 가격 변경이 기존 주문 금액을 바꾸지 않는다.
- 재고 차감 시점은 온라인은 결제 확정, 오프라인은 등록 확정 시점이 기본이다.
- 주문 취소 시 재고 복구 규칙은 명시적으로 설계한다.

---

## 4. 외부 도메인 계약

### 4.1. `member`

- 주문 생성 직전에 `memberId` 기준으로 배송지 원본을 조회한다.
- 거래 완료 이후 배송지 정보는 `store`가 자체 스냅샷으로 보존한다.

### 4.2. `service`

- 상품, 재고, 주문의 쓰기 책임은 `service`가 아니라 `store`가 가진다.
- 관리자 포털이 통합 화면을 제공하더라도 주문 상태 변경은 `store`에서 처리한다.

### 4.3. 통합 조회

마이페이지나 운영 포털은 주문 상태를 집계해 보여줄 수 있지만, 조회 통합이 곧 쓰기 책임 통합을 의미하지는 않는다.

---

## 5. 대표 유스케이스

1. 회원이 주소록에서 배송지를 선택한다.
2. `store`가 결제 직전 배송지 원본과 상품 가격을 읽는다.
3. 결제 완료 시 `Order`, `OrderItem` 스냅샷을 생성한다.
4. 관리자가 배송 라벨 추출 후 발송 처리한다.

---

## 6. 구현 초점

- 주문 스냅샷 고정
- 재고 차감과 취소 시 복구 규칙
- 온라인, 오프라인 채널별 상태 전이
- 배송지 엑셀 추출과 발송 상태 변경

---

## 7. 후속 상세화 항목

- 외부 결제사 연동 프로토콜
- 주문 취소, 환불 상세 정책
- 배송 라벨 출력과 엑셀 서식 규격
- 재고, 주문 운영 리포트 강화
